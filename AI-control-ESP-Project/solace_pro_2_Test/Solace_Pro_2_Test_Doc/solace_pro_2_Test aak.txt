1) Intiface Central / GitHub – Releases & Features
────────────────────────────

- Intiface Central v2.6.4 release note includes: **“Solace Pro Linear Movement Support”**. :contentReference[oaicite:0]{index=0}  
  Dus vanaf v2.6.4 is er ondersteuning voor lineaire beweging (“position / funscript” style) toegevoegd. :contentReference[oaicite:1]{index=1}

- In v2.6.5 en latere releases: updates naar Buttplug v9.0.6 etc., maar kernfeature “solace pro linear movement” blijft aanwezig sinds 2.6.4. :contentReference[oaicite:2]{index=2}

- GitHub repo “buttplugio/buttplug”, issue #606: discussie waarin men zegt:
  > “The Solace Pro implements exactly what The Handy's WiFi API does, except over bluetooth. You can load funscripts to the device, and control them …” :contentReference[oaicite:3]{index=3}

────────────────────────────
2) EroScripts Community Threads
────────────────────────────

- Thread “Intiface Central v2.6.4 out w/ Lovense Solace Pro Linear (Funscript) support” — meldt dat in v2.6.4 Solace Pro nu linear/funscript ondersteuning heeft. Gebruikers die het al gebruikten op v2.6.3 of ouder moeten het device “Forget Device” doen in Intiface, opnieuw starten, weer koppelen, etc., zodat het apparaat met position/oscillation capabiliteiten verschijnt. :contentReference[oaicite:4]{index=4}

- “Solace Pro Funscript Problems” — klachten over traagheid of onnauwkeurigheid bij kleine bewegingen. Bijvoorbeeld gebruikers zeggen dat oscillatie-functie of automatische oscillatie interfereren met fijnere position control door funscript. In één post:
  > “You need to disable oscillation and only use the linear/position actuator.” :contentReference[oaicite:5]{index=5}

- Threads over “Linux Scripts Player for Solace Pro” – iemand bouwt zelf funscript player op Linux; gebruikt Intiface + buttplug + “monsieursoFunscriptPlayer”. Problemen: rauwe bewegingen aan begin (“shakes for ~3 seconds”) etc. :contentReference[oaicite:6]{index=6}

- Thread “Help with setting up Solace Pro for OSF+MultiFunPlayer” – gebruiker verbonden met Intiface; testcontrols werken; funscript geladen in MFP; maar ondanks alles beweegt het apparaat niet (geen daadwerkelijke lineaire beweging), alleen oscilleren. Suggesties om mapping, axes, of instellingen te checken. :contentReference[oaicite:7]{index=7}

────────────────────────────
3) Technische (beperkende) details & bugs
────────────────────────────

- In issue “Lovense Solace is not working with Scriptplayer” (Buttplug forum): verbinding met device, script geladen, heatmap ok, maar apparaat reageert niet op scriptbeweging—alleen op snelheid. Gebruikers melden dat Solace Pro **geen position commands** leek te accepteren vóór de update; alleen snelheid (“ScalarCmd”). :contentReference[oaicite:8]{index=8}

- Vanuit “Solace Pro Funscript Problems”: kleine bewegingen zijn vaak niet merkbaar; snelheid van updates (“upload rate”) en update-modus (“FixedUpdate” vs “PolledUpdate”) hebben effect op responsiviteit. Met name “PolledUpdate” + hoge upload rate helpt. :contentReference[oaicite:9]{index=9}

- Mapping issues: in MultiFunPlayer / VAM / ScriptPlayer moeten axis mapping, auto-home etc. correct ingesteld zijn. Soms heeft “oscillation” aan/uit invloed. :contentReference[oaicite:10]{index=10}

────────────────────────────
4) Samenvatting / Conclusies uit de gebruikersdata
────────────────────────────

- Sinds Intiface Central v2.6.4 werkt Solace Pro met **lineaire beweging / funscript** (position control), als correct verbonden en geconfigureerd.  
- Voor gebruikers: indien Solace Pro al gekoppeld was vóór v2.6.4, dan “Forget Device” + herstart nodig om de nieuwe capabiliteit te zien. :contentReference[oaicite:11]{index=11}  
- Voor fijnere script-bewegingen: oscillation uitzetten, update-modus aanpassen, mapping controleren. Sommige bewegingen worden genegeerd of nauwelijks merkbaar als snelheid/oscillatie laag is. :contentReference[oaicite:12]{index=12}  
- Bepaalde software (ScriptPlayer, MultiFunPlayer, VAMSync, etc.) hebben variërende mate van compatibiliteit; in veel gevallen is de integratie niet perfect (snelheidsveranderingen werken beter dan absolute position veranderingen).  

────────────────────────────
5) Open issues / wat nog niet (goed) werkt
────────────────────────────

- Sommige scripts reageren nog steeds niet met beweging, alleen op snelheid. Mogelijk omdat de client software (ScriptPlayer e.d.) de juiste Buttplug-V3 / LinearCmd ondersteuning niet volledig heeft geïmplementeerd. :contentReference[oaicite:13]{index=13}  
- Lag / vertraging / onnauwkeurige kleine bewegingen blijven een klacht. Mogelijk Bluetooth latency, update rate, of hardware limieten.  
- Oscillation (automatisch heen/weer pumpen) interfereert soms met position control; moet uit of correct gemapped worden.  
- In Linux-implementaties: minder ondersteuning / tools klaar voor funscript met Solace Pro; gebruikers bouwen zelf oplossingen. :contentReference[oaicite:14]{index=14}
